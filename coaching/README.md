# Coaching briefs and AI-generated analyses

This folder contains:

1. **Coaching briefs** — markdown documents generated by `catalyst-prompt` (or the
   GUI's "Generate brief" action). Each brief is a self-contained context dump
   you paste into an LLM (Claude, GPT, etc.) covering:
   - Car/driver setup (inlined from `lotus/`)
   - Track reference (Garmin's official segments + canonical named corners)
   - Recent session data (sessions, per-lap aggregates, per-segment splits,
     per-corner traces)
   - A "Your task" instruction block telling the LLM how to format its response

2. **AI-generated coaching reports** — markdown files the LLM writes back into
   this folder after analysing a brief. Filename convention:
   `<YYYY-MM-DD>-<topic>.md`.

## Generating a brief

```bash
# Three most-recent sessions, default "overview" scope
catalyst-prompt --last 3

# Two specific sessions, with per-corner trace data included
catalyst-prompt --session <guid1> --session <guid2> --scope corner

# Everything, comparison-focused
catalyst-prompt --all --scope compare
```

Output lives at `coaching/<YYYY-MM-DD>-<scope>-brief.md` by default.

## Sending it to an LLM

1. Open the generated brief.
2. Paste the whole thing into your LLM of choice (Claude Desktop, Claude Code,
   GPT, etc.). At ~80–200 KB it fits comfortably in modern context windows.
3. The brief's "Your task" section asks the LLM to write its analysis to
   `coaching/<date>-<scope>.md` — let it propose the contents and either
   accept or paste them into a new file here.

## Conventions for AI-generated reports

Reports should follow the structure the brief prescribes:

```markdown
# <Config> coaching — YYYY-MM-DD

## Headline
## Per-segment analysis
## Per-corner analysis
## Cross-lap consistency
## Cross-session trends
## Recommendations (prioritised)
## Drills
```

Reference specific data: lap numbers, segment IDs, `dist_idx` ranges. If a
report makes a recommendation without citing data, treat it as low-confidence.

## Note on field labels

`f4`..`f15` in the sample data are not yet definitively labeled (Garmin doesn't
ship a `.proto` schema). The brief includes provisional labels based on
value-range analysis. Trust **relative trends** between laps and samples more
than absolute units until the labels are pinned down.
